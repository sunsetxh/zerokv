# ZeroKV 第一次会议纪要

**日期**: 2026-03-03
**阶段**: 需求澄清
**参与者**: PM, Arch, Dev, QA, Coord

---

## 1. 需求背景

设计并实现一个高性能分布式KV存储库：
- 底层通信：UCX (Unified Communication X)
- 扩展能力：HCCL (华为集合通信库) + NCCL (NVIDIA集合通信库)
- 上层接口：C++ 和 Python
- Value规格：1KB ~ 1GB

---

## 2. 需求确认

| 需求项 | 结论 | 备注 |
|--------|------|------|
| 底层通信 | UCX 1.19.0 | - |
| RDMA类型 | RoCE | 不支持InfiniBand |
| 扩展能力 | HCCL + NCCL | 用于集合通信操作 |
| 上层接口 | C++ + Python | - |
| Value规格 | 1KB ~ 1GB | - |
| 使用场景 | AI训练参数共享 | - |
| 一致性 | 强一致性 | 元数据用redis/etcd |
| 持久化 | MVP不支持 | 后续版本考虑 |
| 集群规模 | 16-64节点 | 初期目标 |
| 开源计划 | 是 | - |

---

## 3. 关键技术澄清

### 3.1 UCX配置
- 版本：1.19.0
- 网络：RoCE v2
- 关键配置：UCX_NET_DEVICES, UCX_TLS

### 3.2 特殊场景
- **用户内存**：支持用户注册一段内存，get时直接拷贝到用户内存（零拷贝）

### 3.3 一致性方案
- MVP不使用一致性协议
- 元数据（key路由信息等）持久化到redis/etcd
