## 🔄 文档更新流程

### 每日工作流程

1. **开始工作前**: 查看 `TASKS.md` 确认当前任务
2. **工作中**: 遇到问题在 `WORK_LOG.md` 记录
3. **提交代码前**:
   - 更新 `TASKS.md` 标记任务状态
   - 更新 `WORK_LOG.md` 记录当日工作
4. **完成里程碑**: 更新 `CHANGELOG.md`

### 任务完成检查清单

- [ ] 代码已实现并通过测试
- [ ] `TASKS.md` 已更新（状态改为已完成）
- [ ] `WORK_LOG.md` 已记录（完成情况、遗留问题）
- [ ] 代码已 commit 并推送
- [ ] 如有用户可见功能，更新 `README.md`
